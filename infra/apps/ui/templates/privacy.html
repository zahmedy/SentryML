Data & Privacy

How SentryML handles your data.
What data we receive
SentryML receives prediction events that you send via the API, including:
  • Model identifier
  • Prediction value
  • Optional score or confidence
  • Timestamp
These events are used only to monitor distribution changes over time.
What data we store
  • Prediction events (limited to the fields you send)
  • Aggregated statistics used for drift detection
  • Incident records and timelines
  • Metadata about your models (first seen, last seen, event counts)
We do not modify or enrich your data.
What we do not store
  • Training data
  • Feature values
  • Raw input payloads
  • Model weights or artifacts
  • Ground-truth labels
  • Personally identifiable information (PII), unless you explicitly send it
Data isolation
Each organizationâ€™s data is logically isolated. API keys are scoped to a single organization and are required for all ingestion requests.
API keys are hashed and never stored in plaintext.
Authentication
  • API access is authenticated using API keys
  • Web access uses secure, HTTP-only sessions
  • You can rotate or revoke API keys at any time
Data retention
Prediction events and incidents are currently retained until explicitly deleted. Data retention controls are planned for a future release.
Monitoring behavior
  • Drift detection runs on a scheduled worker
  • Monitoring is enabled per model
  • Incidents resolve automatically when data returns to normal
  • Acknowledging an incident does not silence alerts
Integrations
When enabled, Slack alerts include:
  • Model name
  • Incident severity
  • Summary statistics
No raw prediction data is sent to Slack.
Security
SentryML follows standard security practices for early-stage SaaS products. We do not claim formal compliance certifications at this time.
Questions?
If you have questions about data handling or privacy, contact us before integrating.
